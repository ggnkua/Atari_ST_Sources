
Das CAB-Protokoll
=================

Das CAB-Protokoll besteht aus einer Reihe von AES-Messages mit deren
Hilfe beliebige GEM-Programme bestimmte Aufgaben an CAB Åbertragen kînnen,
bzw. bestimmte Aktionen mit CAB koordinieren kînnen.

Der Inhalt der Environment-Variablen BROWSER= (sofern definiert) enthÑlt
den Zugriffspfad von CAB. Sollte sich CAB nicht im Speicher befinden kann
ein Programm Åber diese Variable CAB selbst nachstarten.

Eine mîgliche Anwendung:
Falls man im Internet weitere aktuelle Informationen oder Updates zu
eigenen Programmen anbietet, kînnte man mit dem CAB-Protokoll ohne groûen
Aufwand den Zugriff auf diese Informationen aus diesen Programmen heraus
ermîglichen. CAB erledigt dann den Download der Daten und die Darstellung.

CAB_HELLO  ($CAB8)  (Nachricht von CAB)
------------------
CAB verschickt nach seinem Start an alle laufenden Programme diese
Nachricht

msg[0] = $CAB8
msg[1] = Id von CAB
msg[2] = 0
msg[3] = Version des CAB-Protokolls ($0100 entspricht V1.00)
msg[4] = Bitvektor mit den von CAB (CAB.OVL) unterstÅtzten Internet-
         Protokollen
             Bit 0: HTTP
             Bit 1: FTP
             Bit 2: GOPHER
             Bit 3: WAIS
             Bit 4: MAILTO
             Bit 5: NNTP
             Bit 6: TELNET
             Bit 7-15: 0 (reserviert)
msg[5] = Bitvektor mit den von CAB unterstÅtzten Nachrichten des
         CAB-Protokolls.
             Bit 0: CAB_VIEW
             Bit 1: CAB_MAIL, CAB_MAILSENT
             Bit 2: CAB_REQUESTSTATUS, CAB_RELEASESTATUS, CAB_STATUS,
                    CAB_EXIT
             Bit 3-15: 0 (reserviert)
msg[6] = 0
msg[7] = 0

--------------------
Wird von CAB an das Programm geschickt, welches CAB_MAIL gesendet hat und

              0: Keine URL angegeben.
              1: Offline, Netzverbindung gestîrt, Server nicht erreichbar.

                 bearbeitet werden sollte, hat der Benutzer die
                 Mîglichkeit die Aktion abzubrechen)
              3: GEMDOS-Fehler beim Zugriff auf die Maildatei.
              4: Editor zum Bearbeiten konnte nicht aufgerufen werden.

msg[6] = 0
msg[7] = 0
