Arbre d'Objet :
-----------------------
Chaque fichier RSC peut contenir un certain nombre d'arbre d'objet.


*rsh_object* contient un offset (par rapport au debut du fichier) qui indique
l'emplacement de la liste objet OBJECT (consécutivement).

Le tableau de 32bits pointé par *rsh_trindex* peut źtre utilisé pour séparer les
arbre d'objet dans la liste. Chaque élément du tableau pointe sur un dial/menu.
Il y a rsh_ntree index dans ce tableau.

rsh_trindex -> tab_long[rsh_ntree]

Aprčs avoir été chargé en mémoire par rsrc_load(), les membres ą rsh_trindex
sont remplacé par un pointeur vers l'adresse réel de ces arbres d'objet (dial/menu).


***************************************************************************************
*** Les objets doivent źtres triés ****************************************************
***************************************************************************************
TOS EmuTOS MyAES XaAES ne suivent pas l'arbre des objets pour afficher la barre
de MENU ni pour calculer le nombre d'objet.
RSC Builder suit l'arbre des objets, il ne fait pas ob_tail-ob_head pour avoir 
le nombre d'objet, mais parcours l'arbre et compte les objets...

RSC Builder, avec l'option Merge_Option_menu, ajouter les objets ą la fin du dial
et modifie le chainage des objets...
Bon... RSC Builder doit trier Le MENU pour que ēa fonctionne sous ces AES...

	Stune_fr.rsc, ztask.rsc, zview.rsc, pareil.

* rsh_string Contains an offset from the beginning of the file to the string data.
	toutes les free_string commence ici...
