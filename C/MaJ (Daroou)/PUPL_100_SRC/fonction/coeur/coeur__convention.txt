Petit essai de normalisation du code source...


Toutes mes applications GEM sont basées sur les fonctions du Coeur (fonction\coeur\...)
(sauf les applis préhistorique qui ne sont plus public maintenant)


L'objectif est que le Coeur des applications s'occupent de tout au maximum du possible,
et que l'application n'ait à gérer en local uniquement les fonctions local pour le but
qu'elle a été créée.


Cette convention a été adopté après une certaine version du coeur, c'est pour cela qu'il reste
beaucoup de variable / define (voir fonction) qui n'ont pas encore ce prefix...
C'est pénible a changer, mais avec le temps... quoique, il me reste plus beaucoup de temps sur
cette planète...




 Fonction ComMune aux applis GEM
 -------------------------------

- Fcm_xxxxxx : Identificateur pour tout ce qui est du domaine du Coeur des applications
               Variables / fonctions / etc...
			   Chaque Fcm_xxxxx est accessible par l'application.

- FCM_xxxxxx : #defines globales du coeur.



- Fcmgfx_xxx : Identificateur pour tout ce qui est du domaine Graphique des applications
               Elles ne font pas partie du Coeur des applications.
               Variables / fonctions / etc...
			   Chaque Fcmgfx_xxxxx est accessible par l'application, doit être include
			   dans le code source.

- Fcmsnd_xxx : Identificateur pour tout ce qui est du domaine Audio des applications




APP_xxxx : define globale de l'application necessaire pout le coeur (prefix titre fenetre par exemple)
app_xxxx : variable globale de l'application necessaire pour le coeur
