----------------------------------------------------------------------------
* supression surface.offscreen_mode (membre de struct SURFACE).
  Outre le fait que le nom de la variable n'avait rien a voir avec sa fonction
  (choix du type de RAM pour la surface). Elle a ete remplace par une variable
  global uint16 Fcmgfx_CreateSurface_ram_type_select (voir fcm_create_surface.h).
  Toutes les surfaces sont alloue dans un meme type de RAM, AUTOmatiquement, ou
  dans un type de RAM choisi par l'utilisateur (fenetre options -> choix RAM Type)
  Si le profil (mode AUTO) choisi un mauvais type de RAM (par bug ou car une
  extension améliore les perfs d'un autre type de RAM), l'utilisateur peut forcer
  un autre type de RAM.

  fonctions graphiques (Fcm_make_xxxxx). Comme seul le FrameBuffer peut etre ouvert
  en mode VDI Offscreen (v_opnbm()), l'affectation de cette variable est automatique
  dans les fonctions de sprite. Plus besoin d'avoir x fois la meme affectation dans
  les fonctions init_surface avec le meme parametre...
