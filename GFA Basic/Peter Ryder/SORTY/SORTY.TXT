Peter Ryder                                       Dezember, 1987

SORTY - ein schnelles, flexibles, universell anwendbares Sortier-
                 vefahren als GFA-Basic-Procedure


Mit dem hier vorgestellten Verfahren können Gleitkommazahlen, In-
tegerzahlen oder Zeichenketten mit dem gleichen Procedure sortiert 
werden. Der Aufruf lautet: 
                        @Sorty(*A(),*B())
A() ist ein eindimensionales Feld, das die zu sortierenden Werte 
enthält und daher vom Typ Var(), Var%() oder Var$() sein kann. B() 
ist ein Integerfeld, das die gleiche Dimension wie A() haben mu×. 
Nach dem Ausführen enthält B() die Indizes des A-Feldes in der 
Reihenfolge aufsteigender A-Werte, also A(B(0)) ist der kleinste 
Wert, A(B(1)) der zweitkleinste Wert usw. Das Feld A() selbst wird 
nicht verändert. Der Vorteil dieses Verfahrens liegt darin, da× 
alle Werte, die zum gleichen Datensatz gehören und daher mit dem 
gleichen Index angesprochen werden, automatisch mitsortiert wer-
den. Ferner können die Datensätze nach verschieden Werten sor-
tiert werden, ohne da× die vorangegangen Sortierungen verloren 
gehen. Es mu× lediglich für jeden Sortiervorgang ein neues Feld 
B() defininiert werden. Ein kleines Beispiel: 25 Sportler nehmen 
an einem Wettbewerb teil, der sich aus mehreren Einzelwettkämpfen 
zusammensetzt (z.B. Zehnkampf). Die Ergebnisse sollen nach Wett-
kampfsnummer, Name, Einzelpunkten und Gesamtpunkten sortiert 
werden. Dies könnte etwa so aussehen:

Dim Name$(24), Wett.nr%(24), Punkte(24), Ges.punkte(24)
Dim Name_ord%(24), Wett.nr_ord%(24), Punkte_ord%(24)
Dim Ges.punkte_ord%(24)
...
@Sorty(*Name$(),*name_ord%())
@Sorty(*Wett.nr%(),*Wett.nr_ord%())
@Sorty(*Punkte(),*Punkte_ord%())
@Sorty(*Ges.punkte(),*Ges.punkte_ord%())

Die Ausgabe aller Daten in der Reihenfolge der Gesamtwertung könn-
te dann mit folgender Schleife erreicht werden:

For J%=24 Downto 0
   I%=Ges.punkte_ord%(J%)
   Print Wett.nr%(I%),Name$(I%),Punkte(I%),Ges.punkte(I%)
next J%

Die Zeichenketten werden alphabetisch sortiert unter Berücksichti-
gung der deutschen Buchstaben ä, ö, ü und ×. Zwischen Gro×- und 
Kleinbuchstaben wird nicht unterschieden.

Als Sortieralgorithmus wird das sog. Binärbaumverfahren verwendet 
(s. z.B. Daniel Thalmann: "MODULA-2, An Introduction", Springer-
Verlag, 1985, S.234). Dieses Verfahren benötigt nur zwei Durch-
gänge und ist deshalb besonders schnell. Im ersten Durchgang wer-
den die Daten in eine Baumstruktur gebracht, und im zweiten Durch-
gang wird der Baum abgesucht, um die gesuchte Reihenfolge zu be-
stimmen. Ein Testprogramm (s. Listing) zeigt, da× das vorliegende 
Programm in der kompilierten Fassung etwa 2 bis 3 Sekunden 
braucht, um 1000 numerische Werte zu sortieren. Wegen der erwähn-
ten Berücksichtigung der deutschen Buchstaben braucht das alpha-
betische Sortieren etwas länger. Die benötigte Zeit steigt etwa 
linear mit der Anzahl der Datensätze an. Der Interpreter braucht 
übrigens die fünffache zeit!

Einige Bemerkungen zum Programm selbst: Im Hauptteil von Sorty
wird zunächst der Typ der zu sortierenden Variablen mit Hilfe des 
Befehls Type ermittelt. Die Zuordnung der Feldvariablen und der 
Aufbau des Baumes erfolgen getrennt nach Variablentyp in ver-
schiedenen Procedures (Sort_string, Sort_real und Sort_int), die 
jedoch viele Befehle gemeinsam haben. Diese programmtechnisch 
nicht sehr elegante Lösung wurde gewählt, um mehrfache Abfragen 
nach dem Variablentyp während des Sortierverfahrens und die damit 
verbundenen Zeitverluste zu vermeiden.

Die Procedures Add_string, Add_real und Add_int erzeugen die Baum-
struktur, wobei von der Möglichkeit des rekursiven Abrufs Gebrauch 
gemacht wird. Das Unterprogramm Tausch sucht eine Zeichenkette in-
nerhalb eines Strings und ersetzt sie durch eine andere, die nicht 
die gleiche Länge haben mu×. Hiermit werden ä, ö, ü, × durch a, o, 
u, ss ersetzt, um sie beim Sortieren richtig behandeln zu können.

Das Procedure Out sucht den Baum ab und gibt die Indizes in der 
gewünschten Reihenfolge aus.



                         Programmlisting


' Testprogrammm - prüft die Geschwindigkeit des Sortierverfahrens.
'
Dim Zeichen$(58)
For I%=0 To 25
  Zeichen$(I%)=Chr$(65+I%)
  Zeichen$(I%+26)=Chr$(97+I%)
Next I%
Zeichen$(52)="ä"
Zeichen$(53)="Ä"
Zeichen$(54)="ö"
Zeichen$(55)="Ö"
Zeichen$(56)="ü"
Zeichen$(57)="Ü"
Zeichen$(58)="×"
Do
  Input "Anzahl der Werte (0 = Schlu×)";N%
  Exit If N%<=0
  M%=N%-1
  Dim A$(M%),A(M%),A%(M%),Ord%(M%)
  For I%=0 To M%
    A$(I%)=""
    For J%=0 To Random(20)
      A$(I%)=A$(I%)+Zeichen$(Random(59))
    Next J%
    A(I%)=Pi*Random(100000)
    A%(I%)=Random(100000)
  Next I%
  T1=Timer
  @Sorty(*A$(),*Ord%())
  T2=Timer
  Print N%'"Strings sortiert in"'(T2-T1)/200'"s"
  T1=Timer
  @Sorty(*A(),*Ord%())
  T2=Timer
  Print N%'"FP-Zahlen sortiert in"'(T2-T1)/200'"s"
  T1=Timer
  @Sorty(*A%(),*Ord%())
  T2=Timer
  Print N%'"Integer-Zahlen sortiert in"'(T2-T1)/200'"s"
  Erase A$()
  Erase A()
  Erase A%()
  Erase Ord%()
Loop
' Sortierverfahren für String-, Gleitkomma- oder Integerwerte
' .................................Peter Ryder, Dezember 1987
'
' Hauptteil des Sortier-Unterprogramms.......................
'
Procedure Sorty(Wert.pt%,Ord.pt%)
  T%=Type(Wert.pt%)
  If T%<4 Or T%>6 Then! Typ mu× var(), var%() oder var$() sein
    Alert 1,"Type error",1,"OK",But%
  Else
    On T%-3 Gosub Sort_real,Sort_string,Sort_int   !Verzweigung
    Erase Left%()
    Erase Right%()
    Erase Order%()
  Endif
Return
'
' Sortieren von Gleitkommazahlen.............................
'
Procedure Sort_real
  Swap *Wert.pt%,Arry()
  Dm%=Dim?(Arry())-1
  Dim Left%(Dm%),Right%(Dm%),Order%(Dm%)
  Arrayfill Left%(),-1
  Arrayfill Right%(),-1
  I%=0
  Do
    Inc I%
    Exit If I%>Dm%
    @Add_real(I%,0)
  Loop
  I%=0
  @Out(0)
  Swap *Wert.pt%,Arry()
  Swap *Ord.pt%,Order%()
  Erase Arry()
Return
Procedure Add_real(Index%,Node%)        ! erzeugt Baumstruktur
  If Arry(Index%)<=Arry(Node%) Then
    If Left%(Node%)<0 Then
      Left%(Node%)=Index%
    Else
      @Add_real(Index%,Left%(Node%))
    Endif
  Else
    If Right%(Node%)<0 Then
      Right%(Node%)=Index%
    Else
      @Add_real(Index%,Right%(Node%))
    Endif
  Endif
Return
'
' Sortieren von Stringvariablen.........................
' Alphabetisch, unter Berücksichtigung von ä, ö, ü und ×
'
Procedure Sort_string
  Swap *Wert.pt%,Arry.$()
  Dm%=Dim?(Arry.$())-1
  Dim Left%(Dm%),Right%(Dm%),Order%(Dm%),Arry$(Dm%)
  Arrayfill Left%(),-1
  Arrayfill Right%(),-1
  I%=0
  Do
    X$=Upper$(Arry.$(I%))   !Gro×/kleinschreibung ignoriert.
    @Tausch(*X$,X$,"Ä","A")       ! Ä -> A
    @Tausch(*X$,X$,"Ö","O")       ! Ö -> O
    @Tausch(*X$,X$,"Ü","U")       ! Ü -> U
    @Tausch(*X$,X$,"×","SS")      ! × -> SS
    Arry$(I%)=X$
    Inc I%
    Exit If I%>Dm%
  Loop
  I%=0
  Do
    Inc I%
    Exit If I%>Dm%
    @Add_string(I%,0)
  Loop
  I%=0
  @Out(0)
  Swap *Wert.pt%,Arry.$()
  Swap *Ord.pt%,Order%()
  Erase Arry$()
  Erase Arry.$()
Return
Procedure Tausch(Str_ptr%,Str$,Del_str$,Ins_str$)
  ' ersetzt Del_str$ durch Ins_str$ in Str$
  Stelle%=1
  Do
    Stelle%=Instr(Stelle%,Str$,Del_str$)
    Exit If Stelle%=0
    Xxx$=Left$(Str$,Stelle%-1)+Ins_str$
    Str$=Xxx$+Right$(Str$,Len(Str$)-Stelle%-Len(Del_str$)+1)
  Loop
  *Str_ptr%=Str$
Return
Procedure Add_string(Index%,Node%)    ! Erzeugt Baumstruktur
  If Arry$(Index%)<=Arry$(Node%) Then
    If Left%(Node%)<0 Then
      Left%(Node%)=Index%
    Else
      @Add_string(Index%,Left%(Node%))
    Endif
  Else
    If Right%(Node%)<0 Then
      Right%(Node%)=Index%
    Else
      @Add_string(Index%,Right%(Node%))
    Endif
  Endif
Return
'
' Sortieren von Integerzahlen.............................
'
Procedure Sort_int
  Swap *Wert.pt%,Arry%()
  Dm%=Dim?(Arry%())-1
  Dim Left%(Dm%),Right%(Dm%),Order%(Dm%)
  Arrayfill Left%(),-1
  Arrayfill Right%(),-1
  I%=0
  Do
    Inc I%
    Exit If I%>Dm%
    @Add_int(I%,0)
  Loop
  I%=0
  @Out(0)
  Swap *Wert.pt%,Arry%()
  Swap *Ord.pt%,Order%()
  Erase Arry%()
Return
Procedure Add_int(Index%,Node%)        ! erzeugt Baumstruktur
  If Arry%(Index%)<=Arry%(Node%) Then
    If Left%(Node%)<0 Then
      Left%(Node%)=Index%
    Else
      @Add_int(Index%,Left%(Node%))
    Endif
  Else
    If Right%(Node%)<0 Then
      Right%(Node%)=Index%
    Else
      @Add_int(Index%,Right%(Node%))
    Endif
  Endif
Return
'
' Absuchen des Baums - gilt für alle drei Typen......
'
Procedure Out(Node%)
  If Node%>=0 Then
    @Out(Left%(Node%))
    Order%(I%)=Node%
    Inc I%
    @Out(Right%(Node%))
  Endif
Return
