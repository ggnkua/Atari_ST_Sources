* ICE PACKER 2.0
----------------
Documentation française (c) Pressimage pour le 3615 STMAG.

Ce programme est un Packer dont les sources ont été placés en Domaine Public par son auteur. Il va vous permettre de compresser des fichiers de données ou des exécutables (dans ce cas, les programmes compactés se décompacteront automatiquement au lancement). Ice Packer utilise 
des algorithmes venant de l'Automation Packer, mais largement améliorés et débuggés. Il est donc à la fois plus puissant et plus rapide à décompacter que tous les autres (Automation Packer, Jek 
Packer, Jam Packer, etc).

Vous trouverez dans cette archive les fichiers suivants :
ICE_2_0.PRG
    Le Packer lui-même.
ICE_UNPA.S
    Le source de décompactage normal.
ICE_UNP2.S
    Le source de décompactage "confortable".
ICE_READ.ME
    La documentation originale.

Voici les possibilités du Packer:

* Compacte tous les fichiers sans erreurs.
* Compacte mieux que l'Automation 2.31.
* Décompacte plus vite que l'Automation 2.31.
* Fonctionne sur disques durs et RAM Disques.
* Possibilité d'utiliser des Jokers (*.PRG, A*.*, ZA*.MOD...)
* Les fichiers sont sauvés avec le nom de l'original.
* La valeur d'Offset la plus efficace est utilisée ($1528).
* Le fichier compacté‚ est vérifié automatiquement.
* Le fichier compacté n'est sauvé que s'il est plus petit que l'original.
* Les données peuvent être extraites à une adresse impaire.

Version 1.1:
* Compression plus rapide.

Version 1.12:
* Si vous appuyez sur [ALTERNATE] durant le compactage, le fichier courant est abandonné, et on passe au suivant.

Version 1.13:
* Certains fichiers compactés avec ICE posaient des problèmes sur STe et avec les nouvelles versions du TOS. C'est réparé!

Version 2.00:
* Utilisation du GEM pour le Packer.
* Si vous ne donnez aucun nom de fichier dans le sélecteur d'objet, tous les fichiers seront compactés.
* Si vous n'aimez pas la souris, appuyez sur [ALTERNATE] durant le chargement. Vous pourrez alors utiliser les commandes clavier.
* Si vous compressez des images, vous voudrez peut-être utiliser le nouvel algorithme de compression d'images. Il comprime environ 2K de plus. Le décompactage est réalisé automatiquement avec ICE_UNPA.S ou ICE_UNP2.S. Si un fichier est compacté avec cet algorithme, un P> est affiché. Dans le cas contraire (pour tous les autres fichiers), un -> s'affichera. Si vous ne compactez jamais d'images, vous pouvez effacer les quelques dernières lignes du source de décompression.
* Avec le source ICE_UNPA.S, vous pouvez décompacter les données à n'importe quelle adresse (par exemple: source = $32345, destination = $72213). Vérifiez bien que l'adresse source est au moins 120 ($78) octets plus loin que l'adresse destination. Dans le cas contraire, vous pourriez écraser les données compactées.
* Le source ICE_UNP2.S décompactera n'importe quel fichier sur lui-même. Le programme a besoin de 120 octets de mémoire libre pour sauver et restaurer les données.
