12.14 VME-Bus Interface


VME-Bus am Falcon

Hier wurden mir ein paar Informationen über den VME-Bus Adapter der Firma Rhothron übergeben, es ist nicht viel, zeigt aber zumindest das es auch am Falcon einen VME-Bus gab. Hersteller ist rhothron.

Die Baugruppe

Es handelt sich bei dem VME-Adapter um eine Baugruppe die den Falcon um einen normgerechten VME-Bus erweitert. Es ist hierbei einer oder mehrere VME-Steckpätze möglich. Der Falcon kann mit dieser Zusatzhardware leicht mit externer Hardware erweitert werden. Durch den Anschluß eines VME- Steckplatzgehäuses kann der Adapter fast beliebig viele Karten betreiben.

Das Interface

Das Interface der VME-Erweiterung wird auf den internen Expansionsbus des Falcon gesteckt. Alle Signale sind voll gepuffert und werden über zwei Flachbandkabel aus dem Falcongehäuse herrausgeführt. Am Ende dieser Flachbankabel befindet sich eine Adapterplatine mit einem normgerechten 64-poligen VME-Stecker. Das Interface buffert alle Daten- und Adressleitungen so das keine Störungen in den Rechner gelangen können. Neben der Bus- und Adresslogik kann auf diesem Adapter auch eine Bus-Arbitrierungslogik implementiert werden. Der so zur Verfügung stehende VME-Bus kann dadurch Multimasterfähig werden. Das Interface und der Adapter werden aus dem Rechnernetzteil versorgt, es ist kein gesondertes Netzteil fällig.

Eine Programmierung oder ein Treiber ist für dieses Interface nicht Notwendig da dieses VME-Interface eben nur einen "Buffer" darstellt. Alle VME-Erweiterungen sind "Memorymapped" und können daher wie normaler Speicher adressiert werden.
