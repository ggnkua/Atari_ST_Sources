19.1 SPIN! CD Driver


SPIN! - CD Driver


Mit SPIN! ist in der Version 0.34 eine weitere Möglichkeit entstanden günstig und z.B auf den HDDRiver basierend ein SCSI CD-ROM Laufwerk am Atari Falcon F030 zu betreiben. Weiterere kommerzielle CD-Treiber werden so nicht mehr benötigt. SPIN! ist ein Treiberpaket für CD-ROMs, zur Zeit bestehend aus dem MiNT-XFS SPIN.XFS (eXtended File System), dem MagiC-XFS SPIN-MAGX.XFS und dem MetaDOS-BOS-Treiber SPIN_SD.BOS, der wiederum auf dem SCSIDRV-Interface von Steffen Engel aufsetzt.(!nl)

Unter dem Filenamen SPIN034.TOS sollten Sie das Archiv überall dort finden können wo Sie sonst gepflegte Atari Softwarebestände finden. Die aktuelleste Release und die neuesten Informationen dazu können Sie jederzeit aus dem Internet unter der Homepage von:

  • Julian Reschke
     

direkt zu Ihrem Ortstarif beziehen.

SPIN! stellt einen CD-Treiber dar der XFSse für MiNT und MagiC und die passenden MetaDOS-BOS-Treiber für SCSI-Geräte zur Verfügung stellt. Bisher hat SPIN! den Status Giftware. Genauer bedeutet das es frei für neue Benutzter und Leute die gespendet haben ist. Alle anderen Benutzer sollten sich hier nochmals die Lizenzbedingungen durchlesen.

SPIN! ist mehr oder weniger ein MiNT- und MagiC-XFS für die Ansteuerung von CD-ROMs in den Formaten ISO9660, ISO9660/Rockridge, ISO9660/Joliet, ISO9660/Apple, Mac-HFS und Audio-CD inklusive Multisession-Support. Ein MetaDOS-BOS-Treiber für ACSI- und SCSI-CDROMs, setzt auf Steffen Engels SCSI-Treiber

  • Steffan Engels
     

oder dem HDDRiver auf. Die Treiber unterstützt Audiokommandos und bei vielen Geräten auch das digitale Kopieren von Audiodaten.
(Nicht bei IDE-Geräten!!)

Laut Autor läuft SPIN! auf folgenden Systemen: 

Treiber           System

SPIN.XFS          MiNT.
SPINMAGC.XFS      MagiC.
SPIN_SD.BOS       alle TOS-kompatiblen Systeme
                  auf denen der SCSI-Treiber von
                  Steffen Engel oder der HDDriver existiert.
                  (ACSI, TT-SCSI, Falcon-SCSI, Mac-SCSI...).

Das XFS unterstützt verschiedene Dateisystemtypen:

- ISO9660 Level 1 (das "normale" Format)

- ISO9660 mit "langen" Namen (wird zum Beispiel bei Amiga-CDs verwendet)

- ISO9660 mit Rockridge-Erweiterungen (nur ein Teil wird ausgewertet)

- ISO9660 mit Apple-Erweiterungen (Resourcefork, Type und Creator)

- ISO9660/Joliet: das Windows-95-Format (Unicode noch nicht getestet)

- Macintosh-HFS

- Audio-CDs (erfordert audio-fähigen Level-2-Treiber (BOS-Treiber))

SPIN! besitzt keine eigenen Routinen zum Zugriff auf das Gerät. Stattdessen wird die Existenz eines Level-2-Treibers (MetaDOS-BOS-Treibers) vorausgesetzt. Diese strikte Trennung macht es möglich, SPIN! mit jedem vorhandenen Level-2-Treiber zu kombinieren.
